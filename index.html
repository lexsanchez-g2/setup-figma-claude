Setup guide

Figma + Claude Code
for product designers

Connect your Figma files to Claude Code so it can read your actual design system - layers, tokens, components - and push editable frames back into Figma.

Before you start
Claude Pro or Max plan
Figma account (free works)
Mac, Windows, or Linux
Terminal open and ready
The free Claude.ai plan does not include Claude Code. Some Figma MCP write features require a Dev or Full seat on a paid Figma plan.
1
Install Claude Code
1
Open Terminal and run
curl -fsSL https://claude.ai/install.sh | bash
2
Launch it
claude
3
Sign in with your Claude.ai account in the browser that opens.
Common issues
If you get:
Native installation exists but ~/.local/bin is not in your PATH.
Run this, then try claude again:
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
If you get:
Unable to connect to Anthropic services Failed to connect to api.anthropic.com: UNABLE_TO_GET_ISSUER_CERT_LOCALLY SSL certificate error (UNABLE_TO_GET_ISSUER_CERT_LOCALLY)
You are likely behind a corporate proxy or TLS-intercepting firewall. Try these in order:
1.
Point Node to your system CA bundle:
export NODE_EXTRA_CA_CERTS=/etc/ssl/certs/ca-certificates.crt claude
If that path doesn't exist on macOS, try:
export NODE_EXTRA_CA_CERTS=$(security find-certificate -a -p /Library/Keychains/System.keychain > /tmp/certs.pem && echo /tmp/certs.pem) claude
2.
Export the system keychain and point to it directly:
security export -t certs -f pemseq -k /Library/Keychains/System.keychain -o /tmp/system-certs.pem export NODE_EXTRA_CA_CERTS=/tmp/system-certs.pem claude
Option 1 usually solves it. If neither works, ask IT to allowlist *.anthropic.com. More info at code.claude.com/docs/en/network-config.
Windows requires Git for Windows installed first. Run from PowerShell or CMD - no need to run as Administrator.
1
Install Git for Windows, then open PowerShell or CMD and run
irm https://claude.ai/install.ps1 | iex
2
Launch it
claude
3
Sign in with your Claude.ai account in the browser that opens.
No Node.js or npm needed - the native installer is self-contained
2
Set up auto-navigation to workspace (Optional)

Avoid trust prompts by automatically launching Claude Code in a dedicated workspace directory.

1
Create your workspace directory
mkdir -p ~/Desktop/claude-g2
2
Add this alias to your shell config file (~/.zshrc for zsh or ~/.bashrc for bash)
echo 'alias claude="cd ~/Desktop/claude-g2 && ~/.local/bin/claude"' >> ~/.zshrc
3
Apply the changes
source ~/.zshrc
If you use bash instead of zsh, replace .zshrc with .bashrc in the commands above.
For Windows PowerShell, you'll create a profile script that changes directory before launching Claude Code.
1
Create your workspace directory
New-Item -ItemType Directory -Path "$env:USERPROFILE\Desktop\claude-g2" -Force
2
Add a function to your PowerShell profile
Add-Content $PROFILE "`nfunction claude { Set-Location `"$env:USERPROFILE\Desktop\claude-g2`"; & claude.exe }"
3
Reload your PowerShell profile
. $PROFILE
Now typing "claude" automatically navigates to your workspace
3
Connect Figma to Claude Code

This adds the Figma MCP server - the bridge that lets Claude Code read your design files and write back to Figma.

1
In Terminal, run
claude mcp add --scope user --transport http figma https://mcp.figma.com/mcp
2
Restart Claude Code
exit
claude
4
Authenticate Figma
Commonly missed - don't skip this step
1
Open available MCP servers
/mcp
2
Use arrow keys to select Figma, then press Enter
3
A browser window opens - click Allow access to authorise Claude Code to connect to Figma.
If you see a yellow triangle next to Figma it still needs authentication. Once done it shows as "connected" with no triangle.
5
Read your Figma file

Give Claude Code a Figma URL and it will read your layers, colors, spacing, and components.

1
Open your Figma file in the browser and copy the URL from the address bar. On Figma Desktop: click Share and copy the page URL.
2
In Terminal, type
example prompt
Read this file and describe the layout and components: [paste URL]
Claude Code now understands your design system
6
Build something

Prompt Claude Code as you would a developer who has read your entire design system. The more context you give, the better the output.

example prompts
Design a login card using my design system
Build an empty state for the dashboard using the components you found
Claude Code builds and opens a preview on a local server
7
Push to Figma

Send the result back to your Figma file as fully editable layers.

example prompt
Push this to my Figma file: [paste figma page URL]
If you just say "Push to Figma" without a URL it pushes to the first page of your file.
Open Figma - new design appears as fully editable layers
8
Copy an entire app to Figma

Point Claude Code at a running app and it will recreate it in Figma with components, variables, and styles all properly connected.

example prompt
I want to copy the entire app to this Figma file: [paste file link]. Make sure you create all relevant variables, components, and styles. All designs inside Figma should be connected at any cost.
Make sure your app is running locally before sending this prompt - Claude Code needs to be able to access and read it.
Figma now has all designs, components, variables, and styles
